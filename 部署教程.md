## 🌐 为什么要部署到云端？

| 方式 | 只有你自己能看到 | 全世界都能访问 |
|------|----------------|--------------|
| 在你电脑上打开 | ✅ | ❌ 别人看不到 |
| 部署到云端（Vercel） | ✅ | ✅ 有网址就能看！ |

**部署之后你得到什么？**
- 一个专属网址，比如：`https://你的名字.vercel.app`

## 第一步：把代码上传到 GitHub

### 📌 这一步做什么？
把你电脑上的网站代码，"拷贝"一份放到 GitHub 上，这样 Vercel 才能拿到你的代码来部署。

---

### 🔑 第1步：生成 GitHub 访问令牌（Personal Access Token）

> 为什么需要这个？因为 GitHub 不支持用密码上传代码了，需要用"令牌"代替密码。

1. 打开浏览器，登录 https://github.com
2. 点击右上角**头像** → **Settings**
3. 左侧滚动到最下面，点击 **Developer settings**
4. 点击 **Personal access tokens** → **Tokens (classic)**
5. 点击 **「Generate new token」** → **「Generate new token (classic)」**
6. 填写：
   - **Note**（备注）：随便写，比如 `我的电脑`
   - **Expiration**（有效期）：选 **No expiration**（不过期）
   - **勾选 `repo`**（全部勾选）
7. 滚动到最下面，点击 **「Generate token」** 按钮
8. **⚠️ 重要！** 看到一串以 `ghp_` 开头的字符，**立刻复制保存好**（只显示一次！）

> 📸 截图保存这串字符，或者粘贴到记事本里！

---

### 💻 第2步：在电脑上上传代码

1. 按 `Win + R`，输入 `powershell`，按回车
2. 输入以下命令（每行输入完按回车）：

```powershell
# 进入你的项目文件夹（把路径换成你自己的）
cd C:\Users\47910\Documents\Yan_Project\Workbuddy

# 设置你的 GitHub 用户名（换成你自己的用户名）
git config --global user.name "你的GitHub用户名"

# 设置你的邮箱（换成你注册 GitHub 用的邮箱）
git config --global user.email "你的邮箱"

# 初始化 Git 仓库（只需要做一次）
git init

# 把远程仓库地址加进来（换成你自己的仓库地址）
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 添加所有文件
git add -A

# 提交（保存这次修改）
git commit -m "第一次上传"

# 推送到 GitHub（会要求输入用户名和密码）
git push origin main
```

3. 当要求输入密码时：**不要输入登录密码！** 粘贴你刚才复制的 `ghp_` 开头的令牌！

> ✅ 看到类似 `main -> main` 的字样就成功了！

## 第二步：把网站部署到 Vercel

### 📌 这一步做什么？
告诉 Vercel："请用 GitHub 上的代码，帮我把网站发布到互联网上！"

---

### 🌐 第1步：用 GitHub 登录 Vercel

1. 打开浏览器，访问：https://vercel.com
2. 点击 **「Sign Up」** 或 **「Log In」**
3. 选择 **「Continue with GitHub」**（用 GitHub 登录）
4. 授权允许 Vercel 访问你的 GitHub 账号

---

### 📦 第2步：导入你的项目

1. Vercel 登录成功后，点击 **「Add New...」** → **「Project」**
2. 在 **「Import Git Repository」** 里，找到你的仓库，点击 **「Import」**
3. 配置页面：
   - **Framework Preset**：选 `Other`（其他）
   - **Root Directory**：保持默认（`.`）
   - **Build Command**：留空（静态网站不需要构建）
   - **Output Directory**：留空或填 `.`
4. 点击 **「Deploy」** 按钮！

> ⏳ 等待约 1-2 分钟，看到 ✅ 绿色的 **"Congratulations!"** 就成功了！

5. 点击 **「Visit」** 按钮，就能看到你的网站了！🌐

## 第三步：连接数据库（Upstash Redis）

### 📌 为什么需要数据库？
你的网站有"管理后台"，可以新增/修改作品、文章等。这些数据需要存在**云端数据库**，不然每次重新部署数据就没了。

---

### 🗄️ 第1步：在 Vercel 里创建 Upstash Redis 数据库

1. 在 Vercel 项目页面，点击左侧菜单 **「Storage」**
2. 点击 **「Create」** 或 **「→ Create Database」**
3. 在列表中找到 **「Upstash」**，点击它
4. 点击 **「Accept and Create」** 同意授权
5. 配置数据库：
   - **Name**：保持默认即可
   - **Region**（区域）：选 **Singapore**（新加坡，离中国近速度快）
   - **Plan**（套餐）：选 **Free**（免费）
6. 点击 **「Create」** 创建！
7. 创建成功后，点击 **「Continue」**
8. 选择你的项目，点击 **「Connect」**（连接数据库到你的项目）

> ✅ 连接成功后，Vercel 会自动帮你设置好环境变量（数据库的连接信息）

---

## 第四步：把本地数据导入云端

### 📌 为什么需要这一步？
你的电脑里 `data/` 文件夹有数据（作品、文章、工具等），但**云端数据库还是空的**。需要把本地数据"搬运"到云端。

---

### 💻 第1步：获取数据库连接信息

有两种方式：

**方式A（推荐）：通过 Vercel 环境变量**
1. 在 Vercel 项目页面，点击 **「Settings」** → **「Environment Variables」**
2. 找到 `KV_REST_API_URL` 和 `KV_REST_API_TOKEN`
3. 点击旁边的 `...` → **「Edit」** 查看并复制值

**方式B：通过 Upstash 控制台**
1. 打开 https://console.upstash.com
2. 点击你的数据库
3. 复制 **REST URL** 和 **REST Token**

---

### 💻 第2步：在电脑上运行导入命令

打开 PowerShell，输入：

```powershell
cd C:\Users\47910\Documents\Yan_Project\Workbuddy

# 设置数据库连接信息（把值换成你刚才复制的）
$env:UPSTASH_REDIS_REST_URL="这里粘贴你的URL"
$env:UPSTASH_REDIS_REST_TOKEN="这里粘贴你的TOKEN"

# 运行导入脚本
node seed.js
```

> ✅ 看到 **"导入完成！"** 就成功了！

---

### 🔄 第3步：重新部署网站

数据导入后，需要重新部署让网站用上新数据：

1. 回到 Vercel 项目页面
2. 点击 **「Deployments」** 标签
3. 点击最新一条部署右边的 `...` → **「Redeploy」**
4. **取消勾选** "Use existing Build Cache"（不用缓存）
5. 点击 **「Redeploy」**

> ⏳ 等待 1-2 分钟，部署完成后访问你的网站，数据就显示出来了！

## ⚠️ 常见问题 & 注意事项

### ❓ 问题1：访问网站显示 "ERR_CONNECTION_RESET"

**原因**：代码里有错误，导致 Vercel 函数崩溃。

**解决办法**：
1. 检查 Vercel 部署日志（Deployments → 点击最新部署 → 查看日志）
2. 常见原因：环境变量名不匹配（见问题3）

---

### ❓ 问题2：管理后台登录显示"服务器配置错误"

**原因**：云端数据库是空的，没有管理员账号信息。

**解决办法**：运行 `seed.js` 导入数据（见第四步）。

---

### ❓ 问题3：主页/管理后台没有数据

**原因**：环境变量名不匹配！

- Vercel 自动创建的环境变量名是：`KV_REST_API_URL` 和 `KV_REST_API_TOKEN`
- 但代码里读的是：`UPSTASH_REDIS_REST_URL` 和 `UPSTASH_REDIS_REST_TOKEN`

**解决办法**：代码里已经处理了这种情况（兼容两种变量名），**确保你用的是最新代码**。

---

### ❓ 问题4：Vercel 部署失败，提示 "No more than 12 Serverless Functions"

**原因**：Vercel 免费版最多只允许 12 个云函数，你的 `api/` 文件夹里文件太多了。

**解决办法**：
- 把相似功能的 API 合并到一个文件里
- 比如：`works/[id].js` 合并进 `works.js`，用 `req.query.id` 判断

---

### ❓ 问题5：`git push` 时提示需要输入密码

**解决办法**：
- **用户名**：输入你的 GitHub 用户名
- **密码**：**不要输入登录密码！** 粘贴你的 Personal Access Token（`ghp_` 开头）

## 🔧 部署完成后的日常操作

### 📝 修改了网站代码后，怎么更新到线上？

**超简单！** 只需要两步：

```powershell
cd C:\Users\47910\Documents\Yan_Project\Workbuddy

# 添加修改的文件
git add -A

# 提交
git commit -m "描述你改了什么"

# 推送到 GitHub（Vercel 会自动重新部署！）
git push origin main
```

> 🎉 **推送后 Vercel 会自动检测并重新部署**，大约 1-2 分钟后线上网站就更新了！不需要手动操作 Vercel。

---

### 🖼️ 如何在管理后台上传图片？

Vercel 是无服务器环境，**不能直接存图片文件**。有两种方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **方案A：用图床（推荐）** | 简单，免费 | 需要手动上传 |
| **方案B：Vercel Blob** | 自动上传 | 需要写代码对接 |

**方案A操作步骤**（最简单）：
1. 打开 https://imgbb.com 或 https://sm.ms
2. 上传你的图片
3. 复制图片链接（以 `.jpg` 或 `.png` 结尾）
4. 在管理后台的"图片URL"里粘贴这个链接

## 🆘 遇到问题怎么办？

1. **先看 Vercel 部署日志**：Deployments → 点击部署记录 → 查看报错信息
2. **检查环境变量**：Settings → Environment Variables，确认数据库变量存在
3. **重新部署**：有时候是缓存问题，重新部署就能解决
4. **问 AI 助手**：把错误截图发给 AI，描述你做了什么操作
